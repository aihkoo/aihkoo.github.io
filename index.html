COMSOL Server及Client使用

By aihkoo

发表于 Apr 20 2018

Server初始化

本周终于靠着三千预算解决了只差一个服务器的问题，然后开始了漫长的COMSOL Server配置过程。

首先参照COMSOL 系统要求安装两个基础包libwebkitgtk-1.0-0和lsb-core

sudo apt-get install libwebkitgtk-1.0-0 lsb-core

然后就可以按照之前的COMSOL Server配置安装COMSOL Server和初始化配置，就在我以为大功告成之际，残酷的现实告诉我，不肯花钱是要付出时间的。

以下省略一个星期的Baidu和Google时间，因为真是Baidu无门，Google无路。最后发现的问题和解决方案都很简单，5.3a SSQ有bug，滚回5.3。

Client登陆和使用

COMSOL支持两种登陆方式，网页登陆和Client登陆。

网页登陆

直接输入Server地址加端口号，中间用冒号隔开，即可登陆。网页支持管理员账户登陆，管理员可查看Server运行状态，管理其他用户等。

Client登陆

Client需要在官网单独下载COMSOL Client，打开后在Server处输入地址，右侧输入用户名密码即可登陆。

Client和网页端显示基本一样。需要运行仿真时选择Upload，点击Choose applications。选择要仿真的程序并点击Upload，等待上传完成，然后找到上传的仿真程序，Run！

COMSOL Server配置

By aihkoo

发表于 Apr 10 2018

COMSOL Server的安装和COMSOL Multiphysics即单机版本几乎没有区别，且二者使用的镜像文件相同，仅安装时所选许可证文件不同。软件版本为包含SSQ的5.3a，安装步骤参考COMSOL Server 手册，基本上是一路Next，各种选项与手册中推荐一致，区别在于安装之后的配置，以下对Windows版和Linux版分别进行配置。

Windows版(Windows 10)

官方安装默认创建本地管理员用户，但使用SSQ的安装方法并不会创建，也即不会产生%DIR%bin\tomcat\conf\login.properties等文件，使得安装后Server不可用。SSQ中包含的.bat批处理文件运行会报错，需要根据COMSOL安装目录进行修改。
使用记事本或文本编辑工具打开.bat文件，文件共99行，下面以绝对行号对该文件进行修改

管理员权限

8-14行检测是否以管理员身份运行，程序运行时会报错，可以直接删掉，运行程序时以管理员身份运行即可。删掉程序如下： 

REM Detecting if we are administrators  
sfc 2>&1 1>"%TEMP%\testme.txt"
find /i "/SCANNOW" "%TEMP%\testme.txt" >nul
if %errorlevel% NEQ 0 goto no_admin
del /Q "%TEMP%\testme.txt"

安装目录

19-25行检测安装版本和安装目录。作者本来想适配从5.0到5.4b的所有版本，但可能一不小心把5.3打成了5.5，所以5.3是肯定运行不了的……同时，作者还忽略了我等小白会采用系统默认路径C:\Program Files，Program和Files中间的一个空格就把作者的伟大程序给毁了，导致程序怎么运行也不对。不过既然这段程序只是想知道安装版本和安装目录，那可以改成更简单粗暴的形式。原程序如下：

REM We dont expect COMSOL to fix this bug fast...  
for %%a in (50 51 52 52a 52b 55 53a 53b 54 54a 54b) do (   
  if not "x%COMSOLROOT%x"=="xx" goto skip
  for /F "tokens=3" %%b in ('reg query HKLM\SOFTWARE\Comsol\COMSOL%%aServer /v COMSOLROOT 2^>nul ^| findstr "COMSOLROOT"') 
  do set "COMSOLROOT=%%b" && set "COMSOLSERVERVER=%%a"
  :skip
  echo.>NUL
)

由于我的是5.3a版，且安装在了D盘，所以修改程序如下： 

set "COMSOLROOT=D:\Program Files\COMSOL\COMSOL53a\Server"  
set "COMSOLSERVERVER=53a"

配置文件

50-56行是批处理复制文件，作者在这里调用了7z的压缩程序，因此作者提供的exe是必不可少的，需要把作者提供的exe和bat程序放到同一个文件夹下，当然这时候运行还是会出现找不到7z.dll的问题，所以还是装一个7zip软件吧，请自行百度。安装7zip后以管理员身份运行bat文件，等待文件复制完成，然后大功告成，网页或Client登陆。

Linux版(Ubuntu 16.04)

同样也是按照官方手册一路Next，同样也是安装之后死活不能用，同样也是大神作者提供了shell，可惜还是会报错……
不过作者在写shell的时候没那么勤劳，所以需要我们修改的也就没那么多

[ "`uname`" == "Linux" ] && COMSOL_TARS="fl fl_setup_glnxa64 fl_setup_maclnx fl_jvm_glnxa64 fl_glnxa64 mph_maclnx mph_glnxa64"  
[ "`uname`" == "Darwin" ] && COMSOL_TARS="fl fl_setup_maclnx fl_maclnx fl_jvm_maci64 fl_maci64 mph_maclnx mph_maci64"

作者也许想适配不同Linux，可惜失败了，我的是Ubuntu16.04，于是直接修改如下： 

COMSOL_TARS="fl fl_setup_glnxa64 fl_setup_maclnx fl_jvm_glnxa64 fl_glnxa64 mph_maclnx mph_glnxa64"

需要注意的是文件需要放到Server目录下，然后root执行即可，开启服务器一句话搞定

/bin/comsol server


还可以搞一个服务程序，具体可以参考官方手册。

果然还是Linux暴力美学，目前已经熟练掌握了COMSOL Server的安装和卸载，只差一个服务器了。
